# Figma UI/UX Design Descriptions - Personalized Healthcare Assistant

This README describes the Figma UI/UX screens designed for the **Personalized Healthcare Assistant** project. The system is designed to help patients manage health records, medicine reminders, appointments, doctor communication, notifications, and healthcare-related services through a simple web/mobile interface.

---

## Project Name

**Personalized Healthcare Assistant**

---

## Figma Design Purpose

The Figma design represents the user interface flow of a smart healthcare assistant platform. The main goal of the design is to make healthcare management easier for patients, doctors, and caregivers by providing clear navigation, simple forms, useful reminders, and quick access to important health services.

---

## Main Figma Screens and Descriptions

| Screen / Frame Name | Description |
|---|---|
| **Home Page** | The landing page of the healthcare assistant. It gives users quick access to important services such as finding an ambulance, finding a hospital, booking an appointment, viewing specialists, learning about the platform, signing up, logging in, and getting started. |
| **Sign-Up Page** | Allows new users to create an account using basic information such as email and password. It also provides navigation for users who already have an account and want to go to the login page. |
| **Login Page** | Allows existing users to log in using their credentials. It includes options such as “Forget Password?” and “Keep me logged in” for better usability and account access support. |
| **Profile Page** | Displays the user’s personal profile after login. Users can access health records, medicine lists, saved doctors, health tips, and appointment management options from this page. |
| **Dashboard** | The main user dashboard after login. It welcomes the user personally and provides shortcuts to major features such as Live Video Chat, Find Doctor, Nutritionist, Order Medicines, Health Records, Download Report, and Journals. It also shows medication reminders and trust indicators such as BMDC-verified doctors. |
| **Appointment Booking Page** | Allows users to book a doctor appointment by filling in personal details, choosing a doctor, selecting date and time, and uploading required documents. It includes Submit and Reset buttons for form control. |
| **Payment Page** | Used for appointment or service payment. Users can enter card details and confirm payment. It also shows doctor information such as name, degree, department, and contact email. |
| **Confirmation Page** | Appears after successful payment or booking. It confirms the action and provides useful options such as setting an appointment alarm, enabling notifications, and saving the doctor to the user’s doctor list. |
| **Notifications Page** | Shows important reminders and alerts, such as medicine reminders and upcoming appointment alarms. This page helps users stay updated and follow their healthcare schedule properly. |

## Main Design Goals

- Keep the interface simple and easy to understand.
- Make important healthcare features easy to access.
- Use clear buttons and readable text.
- Support elderly and general users with a clean layout.
- Reduce missed medicine and appointment issues through reminders.
- Build trust using verified doctor information.
- Keep the design suitable for both web and mobile platforms.

---

## Accessibility Considerations

The Figma design should consider:

- Large and readable fonts
- Clear contrast between text and background
- Simple navigation
- Easy-to-tap buttons
- Minimal visual clutter
- Consistent layout across pages
- Support for elderly and chronically ill users

## Recommended Prototype Interactions

| From Screen | Action | To Screen |
|---|---|---|
| Home Page | Click **Get Started** | Sign-Up Page |
| Home Page | Click **Log In** | Login Page |
| Sign-Up Page | Submit valid details | Profile Page / Dashboard |
| Login Page | Submit valid credentials | Dashboard |
| Dashboard | Click **Book Appointment** | Appointment Booking Page |
| Appointment Booking Page | Click **Submit** | Payment Page |
| Payment Page | Click **Pay Now** | Confirmation Page |
| Confirmation Page | Click **Notifications** | Notifications Page |
